Linking JS file

Until now, whatever JS we’ve learnt, we did it by writing code in browser console.
But now we’ll see how to create a JS file, how can we link it to HTML file & how by writing code into JS file we can make it visible in the console.
to link the JS file, we’ll use the <script> element in our HTML file.
JavaScript
Copy
<script src=”app.js”></script>
​
Output:
Generally, we don’t mention this <script> tag in head, we mention it just before ending </body>.
Because, JS adds functioning on our webpage and before adding any functioning on the content, that HTML content needs to be loaded first on the webpage.
For loading HTML elements first, we write the HTML code first and at last we mention our JS code.
This is a good practice to always follow.
The comments don’t get printed on the console.
While writing code into browser, we weren’t used to mention semi-colon at the end, but we need to / should mention semi-colon at the end of each statement, basically it gives our editor the signal that this is the end of this programming statement, otherwise it may end up giving errors in some cases.
Now all the messages we’ve printed on the console through app.js, this will get displayed only on the index.html webpage.
Because this app.js file is only associated with index.html webpage.
